# SmartThink

**천재적 사고 엔진 시스템** — [Claude Code](https://docs.anthropic.com/en/docs/claude-code)를 위한 전략적 사고 엔진.

9개 사고 프레임워크(81개 멘탈 모델, TRIZ 발명 원리, 안티프래질 전략 등)와 Cynefin 상황 진단을 결합하여 어떤 주제든 깊이 있는 창의적 분석을 제공합니다.

---

## 빠른 시작

### 1. 설치

```bash
git clone https://github.com/bkan-hq/smartthink.git
cd smartthink
chmod +x install.sh uninstall.sh
./install.sh
```

### 2. 새 Claude Code 세션 열기

```bash
claude
```

> **중요**: 설치 후 **새 세션**을 시작해야 스킬이 인식됩니다.

### 3. 실행

```
/smartthink 1인 기업의 확장 전략
```

주제를 진단하고, 최적의 사고 프레임워크를 선택한 뒤, 10개 이상의 실행 가능한 아이디어가 포함된 분석 결과를 제공합니다.

---

## 3가지 모드

### Deep 분석 (기본값)

9개 레퍼런스 모듈을 로딩하여 메인 에이전트가 직접 분석합니다. 사용자와 인터랙티브하게 방향을 조율합니다.

```
/smartthink AI 스타트업에서 네트워크 효과를 만드는 방법
```

검색 데이터 기반 분석:
```
/smartthink search 2026년 SaaS 시장 트렌드
```

**제공 내용:**
- Cynefin 도메인 진단 (Clear / Complicated / Complex / Chaotic)
- 3개 이상 관점의 다층 분석
- 10개 이상 아이디어 (사용 엔진/모델, 실행 가능성, 기대 효과)
- Top 3 유니콘 잠재력 평가 및 실행 로드맵
- 실행 가능한 Next Steps
- Brief 내보내기 (구현 가능한 주제)

### Agent 분석

서브에이전트(STSA)에게 전체 분석을 위임합니다. Deep과 동일한 방법론이지만 모든 판단을 자동 수행합니다.

```
/smartthink agent 1인 기업의 수익 모델 설계
```

**적합한 상황:** Deep 수준의 분석을 원하지만 인터랙션 없이 자동으로 완료되길 원할 때.

### Light 분석

레퍼런스 모듈 없이 경량 분석. 컨텍스트의 ~5%만 사용합니다.

```
/smartthink light 사이드 프로젝트 아이디어
```

**적합한 상황:** 빠른 브레인스토밍, 간단한 질문, 또는 다른 작업을 위해 컨텍스트를 아끼고 싶을 때.

## 동작 원리

```
입력: /smartthink [모드] 주제
         ↓
    ┌─────────────────────────┐
    │  메인 에이전트            │
    │  1. Cynefin 진단         │
    │  2. 모듈 추천            │
    │  3. 모드별 분기           │
    └──────────┬──────────────┘
               ↓
    ┌──────────┼──────────────┐
    │          │              │
    ▼          ▼              ▼
  Deep       Agent          Light
  (인라인)    (서브에이전트)    (레퍼런스 없음)
  - 모듈 직접  - STSA 위임    - 제1원리
    로딩       - 자동 판단     - ~5% ctx
  - 인터랙션   - 30 turns     - 3-5 ideas
  - 10+ ideas - 10+ ideas
```

**Deep**은 레퍼런스 모듈을 메인 컨텍스트에 로딩하고 사용자와 인터랙션합니다. **Agent**는 서브에이전트(STSA)에 위임하여 자동 분석합니다. **Light**는 레퍼런스 파일 없이 빠르게 분석합니다.

## 진화 시스템

SmartThink는 매 세션에서 학습하며 점점 똑똑해집니다.

**동작 방식:**
- 분석 후 효과적이었던 것과 부족했던 점을 기록
- 효과적인 사고 패턴을 **인사이트**로 저장 (최대 10개 슬롯)
- 사각지대를 **갭**으로 추적 (최대 5개 슬롯)
- 모듈 다양성을 모니터링 — 특정 프레임워크에 과도하게 의존하면 다른 모듈을 권장

**3세션 후 예시:**
```
## 핵심 인사이트 (3/10)
- **AI 레버리지 = 스케일 상한선 제거**: AI가 1인 기업의 매출 상한을 해방
  적용 맥락: 1인 사업에서 팀 규모 제약 극복 시
  근거: 전기가 공장 규모를 해방시킨 것과 동일 구조

## 다양성 H=2.13(>=1.8). 원천: 핵심엔진3, 현실왜곡1, 패턴합성1
```

진화 데이터는 `skill/.data/evolution-state.md`에 저장됩니다. 기기 간 보존:

```bash
cd smartthink
git add skill/.data/evolution-state.md
git commit -m "인사이트 업데이트"
```

## Brief 내보내기

**구현 가능한 작업**(코드, 시스템 설정 등)이 포함된 주제 분석 후 압축된 brief를 자동 생성합니다:

```
skill/.data/briefs/
  └── 2026-03/
      └── 141530-seo-blog-automation/
          └── brief.md          ← Top 3 아이디어 + 실행 계획
```

Brief는 기본적으로 gitignore 처리됩니다 (세션별 분석 결과 포함).

## 사고 모듈

| 모듈 | 핵심 프레임워크 |
|------|---------------|
| 핵심 엔진 | 제1원리, 비대칭 기회, 네트워크 효과, 시장 창조, 모트 구축, 반직관 검증, 가치 포착, 타이밍, 복합 우위, 생태계, 역전 사고, 롤라팔루자 |
| 유니콘 플레이북 | $0→$1B 비즈니스 빌딩 (6단계) |
| 현실 왜곡 | 제약 역전, 카테고리 창조, 시간적 차익거래 |
| 인지 무기고 | 9개 분야 81개 멘탈 모델 |
| 패턴 합성 | 교차 도메인 패턴 인식, 약한 신호 감지 |
| 실행 속도 | OODA 루프, 의사결정 프레임워크, 블리츠스케일링 |
| 안티프래질 전략 | 바벨 전략, 옵셔널리티, 블랙 스완 포지셔닝 |
| TRIZ 혁신 | 40 발명 원리, 모순 해결 |
| 메타인지 | 재귀적 자기개선, Wardley 매핑, 롤라팔루자 |

SmartThink는 주제 유형(아이디어 발굴, 전략, 문제 해결 등)에 따라 자동으로 2-3개 모듈을 선택합니다.

## 분석 파이프라인

### Step 0: 진화 상태 활용

로딩된 진화 상태에 인사이트가 있으면:

1. **핵심 인사이트** — Step 1(모듈 로딩)에서 과거에 효과적이었던 모듈 조합을 우선 검토하라
2. **활성 갭** — Step 2(다층 분석)에서 과거에 부족했던 관점을 의식적으로 보완하라
3. **진화 액션** — 분석 후 사용자에게 보고하라 (분석 흐름은 변경하지 않는다)

- **편향 주의**: 과거 인사이트가 현재 문제에 부적합할 수 있다. Cynefin 진단 결과가 과거 인사이트보다 **항상** 우선한다.
- **다양성 인식**: 인사이트의 원천 모듈 분포를 확인하라 (태그 형식 또는 맥락/근거 줄에서 추론). 한 모듈이 50% 이상이면 과대표. 대안 렌즈를 의식적으로 검토하되, 주제 적합성이 최우선이다.

> **[Deep 모드 확인 가능]** 진화 상태 기반 편향이 의심되면 사용자에게 확인할 수 있다. "과거 인사이트가 [X 방향]으로 편향됩니다. 이번 분석에도 적용할까요?"
> **[Agent 모드]** Cynefin 진단, 분류, 진화 상태를 종합하여 자동 판단.

### Step 1: 선택 모듈 로딩

- 선택된 모듈의 레퍼런스 파일만 Read하라 (레퍼런스 아키텍처 테이블 참조)
- 핵심 엔진 모듈이 선택된 경우: 선택된 엔진에 해당하는 섹션에 집중하라
- 읽은 후 각 모듈의 프레임워크와 운영 절차를 내재화하라

### Step 2: 다층 분석 (최소 2,000자)

**검색 데이터가 있으면** → 검색 최적화 분석:
- **데이터 60% : 프레임워크 40%** 비율 준수
- 먼저 검색 데이터를 체계적으로 정리:
  - 플레이어/대상 프로파일: **최소 10개** (Tier 1/2/3 구분, 정량 지표 포함)
  - 정량 비교 테이블 필수
  - 핵심 트렌드/역학 목록 (데이터에서 발견된 것 중심)
- 그 다음 프레임워크를 적용하되 **2-3개 관점**으로 축소 (5관점 아님)
  - Cynefin 진단: 1줄 헤더만 (확장 설명 불필요)
  - 프레임워크는 데이터를 해석하는 렌즈로만 사용 — 데이터를 대체하지 않는다
- 모든 정량 주장에 **출처 명시 필수**: "시장 규모 $3.5B (Gartner, 2025)"
- 출력 말미에 **출처 부록** 테이블 필수 포함

**검색 데이터가 없으면** → 표준 분석:
- 선택된 모듈 프레임워크를 교차 적용하라
- 최소 **3개 관점**에서 분석하라 (다차원 분석)
- 핵심 엔진의 수학적 모델로 **정량적 판단**
- 인지 무기고 멘탈 모델로 **사각지대** 점검
- 각 분석에서 **반직관적 인사이트**를 의식적으로 탐색하라

> **[Deep 모드 확인 가능]** 분석 중 근본적 방향 분기가 나타나면 사용자에게 확인할 수 있다. "분석 중 [방향 A] vs [방향 B] 분기가 나타났습니다. 어느 쪽을 심화할까요?"
> **[Agent 모드]** 선택된 모듈의 프레임워크 지지도가 더 높은 방향을 자동 선택. 선택 근거를 출력에 기록.

### Step 3: 아이디어 생성

**검색 데이터가 있으면** → **6-8개** 아이디어 (각 아이디어에 **데이터 근거** 필수)
**검색 데이터가 없으면** → **10+개** 아이디어 (최소 3,000자)

각 아이디어에 포함:
- 아이디어 제목 (1줄)
- 핵심 설명 (2-3문장)
- **사용 엔진/모델** (TRIZ 원리 사용 시 원리 번호+이름 명시)
- **데이터 근거** (검색 모드: SEARCH_DATA의 특정 데이터 포인트 인용)
- 실행 가능성 평가 (High/Medium/Low) + 근거
- 기대 효과 (정량 추정 포함)
- 리스크 & 안티프래질 전략

### Step 4: 종합

- **Top 3** 아이디어 선정 + 근거
- 각 Top 아이디어에:
  - **유니콘 잠재력 평가** (TAM, 모트, 타이밍)
  - **실행 로드맵** (30일 / 90일 / 1년)
  - **킬러 가정** (실패를 유발할 핵심 전제)
- 즉시 실행 가능한 **Next Steps** 제안

### Step 4.5: 실전 체크리스트

> 검색 데이터가 있으면 이 단계를 건너뛴다 (데이터 분석에 공간을 할애).

분석 완료 후 아래 3개 섹션을 추가하라. **프레임워크 용어 사용 금지** — 평이한 언어로 작성:

- **즉시 실행**: 이번 주 안에 할 수 있는 구체적 행동 3개
- **절대 금지**: 이 주제에서 흔한 실수 3개 (왜 실수인지 1문장 설명)
- **판단 기준**: 성공/실패를 측정할 핵심 지표 3개 (가능하면 수치 기준 포함)

### Step 5: Brief 내보내기 (조건부)

> Next Steps에 **구현 가능한 작업**(코드 변경, 파일 생성, 시스템 설정 등)이 포함된 경우에만 실행.
> 전략/사고 수준의 Next Steps만 있으면 건너뛴다.

1. 디렉토리 생성: `mkdir -p {DATA_DIR}/briefs/{YYYY-MM}/{DDHHmm}-{topic-slug}/`
   - `{YYYY-MM}`: 현재 연-월 (예: `2026-03`)
   - `{DDHHmm}`: 일(DD) + 시(HH) + 분(mm), 반드시 6자리 (예: `141530` = 14일 15시 30분). 시간 생략 금지.
   - `topic-slug`: 주제에서 핵심 2-4 단어, kebab-case (최대 30자, ASCII 영숫자와 하이픈만). 비영어 주제는 영어로 번역/음역 (예: "AI 스타트업 아이디어" → `ai-startup-ideas`). **검증**: 슬러그에 `[^a-zA-Z0-9-]` 문자가 있으면 `unknown-topic`으로 대체
2. 아래 형식으로 `brief.md` 작성:

```markdown
# SmartThink Brief

## 메타데이터
- 생성일시: {YYYY-MM-DD HH:mm} (시:분 필수, e.g., 2026-03-14 15:30)
- 원본 주제: {topic}
- Cynefin: {diagnosis result}
- 분류: {type}
- 활성 모듈: {module combination}
- 데이터 소스: {search mode status}

## 선정 아이디어 (Top 3)
### 1. {idea name}
- 핵심: {2문장}
- 실행 가능성: {High/Medium/Low}
- 기대 효과: {정량}

### 2. {idea name}
(동일 형식)

### 3. {idea name}
(동일 형식)

## 실행 대상
```

> **압축 원칙**: 원본 5,000+자 → 1,500자 이내. Top 3만, 핵심 인사이트 3-5개만, 메타인지 제외.

### Step 6: 메타인지 및 진화 상태 갱신

분석 후 두 가지 작업을 수행한다: (A) 메타인지 로그를 출력에 포함, (B) 진화 상태 파일 갱신.

**(A) 메타인지 로그** — 출력의 "메타인지 기록" 섹션에 작성:
1. **유용한 모듈/엔진**: 실제로 인사이트를 생성한 프레임워크
2. **부족한 점**: 적용했지만 유용하지 않았거나, 필요했지만 없었던 모듈
3. **새 모델/패턴 발견**: 기존 모듈에 없는 새로운 사고 패턴
4. **롤라팔루자 효과 감지 여부**

**(B) 진화 상태 갱신** — `{DATA_DIR}/evolution-state.md`를 Read 후 아래 프로토콜로 갱신.

#### Semantic Merge Protocol

(A)에서 **인사이트 후보**와 **갭 후보**를 추출한 뒤, 기존 상태 파일과 병합한다.

**인사이트 후보 추출**: "유용한 모듈 조합 + 왜 효과적이었는가"를 1개 인사이트로 요약. 새 패턴 발견 시 별도 후보로 추가.
**갭 후보 추출**: "부족한 점"이나 "필요했지만 없는 프레임워크"가 있으면 1개 갭으로 요약.

**인사이트 병합 — 각 후보를 기존 슬롯과 비교하여 4가지 액션 중 하나 수행:**

| 비교 결과 | 액션 |
|-----------|------|
| **유사** (기존과 70%+ 의미 중복) | 기존 슬롯 텍스트를 **강화** (더 정제된 버전으로 교체). 3줄 한계 유지. |
| **모순** (기존과 반대) | 기존 슬롯을 새 인사이트로 **교체**. |
| **새로움** (모든 기존 슬롯과 이질적) | 빈 슬롯에 **추가**. 10개 모두 찬 경우 가장 범용적/자명한 인사이트를 **축출** 후 추가. |
| **무관** (이 세션 한정 사소한 관찰) | **무시**. 기록하지 않는다. |

**갭 병합:**

| 비교 결과 | 액션 |
|-----------|------|
| **기존 갭과 동일** | 카운트 +1. 카운트 >= 3이면 "진화 액션"에 구체적 액션 추가. |
| **새 갭** | 빈 슬롯에 추가 (count=1). 5개 모두 찬 경우 가장 오래된/카운트 낮은 갭 축출. |
| **해소된 갭** (이 세션에서 해결됨) | 갭 삭제, 슬롯 비움. |

**슬롯 형식 — MANDATORY (이 형식을 정확히 따르지 않으면 다음 세션의 시맨틱 머지가 실패한다):**

> **금지**: 번호 목록(`1.`, `2.`) 사용 금지 — 반드시 불릿(`-`)만 사용한다.

인사이트 — 반드시 불릿 + 3줄. 1줄로 압축 금지:
```
- **[조합/패턴명]**: [핵심 내용 1줄]
  적용 맥락: [어떤 유형의 문제에 효과적인지, 1줄]
  근거: [왜 효과적인지, 1줄]
```

갭 — 반드시 불릿 + 2줄. `(발생 {N}회)` 카운트 필수:
```
- **[부족 영역]** (발생 {N}회): [필요한 것, 1줄]
  상황: [어떤 유형의 문제에서 부족했는지, 1줄]
```

진화 액션 — 반드시 체크박스 형식. 헤더에 직접 서술 금지:
```
- [ ] [구체적 액션]: [어떤 갭이 트리거했는지]
```

**세션 이력 갱신:**
- 최근 5세션 테이블에 현재 세션 정보를 추가한다. 5행 초과 시 가장 오래된 행을 삭제한다.
- 형식: `| {N} | {YYYY-MM-DD} | 주요모듈1, 주요모듈2 | 탐색주제 |`

**다양성 H-score 갱신:**
- 다양성 라인 형식: `## 다양성 H={float}(>=1.8). 원천: {모듈명}{N}, {모듈명}{N}, ...`
- `{N}`은 해당 모듈이 원천인 인사이트 수 (예: `핵심엔진3, 현실왜곡1`). 횟수 없이 모듈명만 나열하면 Shannon entropy 계산 불가 — 반드시 횟수 포함.
- 인사이트의 원천모듈 분포에서 Shannon entropy를 계산하여 H 값을 갱신한다.

**줄 수 검증**: 갱신 후 파일이 60줄 초과 시, 가장 범용적 인사이트 1개를 축출하여 60줄 이하로 맞추라.

**쓰기 방법**: 파일을 Read → 병합 수행 → Write (전체 덮어쓰기). Edit이 아닌 Write 사용.

## 출력 형식

최종 출력은 반드시 아래 형식을 따른다 (한국어):

```markdown
## SmartThink 분석: [주제]

### 핵심 요약 (프레임워크 없는 3줄)
1. [가장 중요한 결론 — 프레임워크 용어 없이 평이한 언어로]
2. [두 번째 결론]
3. [세 번째 결론]
```
